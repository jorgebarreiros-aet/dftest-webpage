DFTest 1.4

Released April 2026  ·  dftest-1.4-all.jar  ·  requires JDK 21+

Introduces a folder-browser dialog for managing source files, support for loading multiple test files simultaneously, and several instrumentation reliability fixes including Allman-style brace handling and a name-collision fix for local variables named cfg.

⬇  Download v1.4
Step 1 — Prerequisites

Make sure you have a JDK 21 (or later) installed. To check, run:

java -version

If not installed, download from Adoptium or any JDK 21 distribution.

Step 2 — Launch

Open a terminal in the folder where you saved the JAR and run:

java -jar dftest-1.4-all.jar

On Windows you can also double-click the JAR if your JDK is on the system PATH.

Step 3 — First use

Once the window opens:

  1. Click 📂 Sources… to browse and load .java files
  2. Click ▶ Analyze to build the CFG
  3. Enter test paths — coverage is evaluated automatically
  4. Use 📊 Matrix to explore test redundancy and minimal cover

To run tests automatically, use 📄 Set test file… to load one or more JUnit 5 test classes. The built-in preset button will locate JUnit jars in your local Maven repository automatically.

Release history
Version 1.4 Current
April 2026 ⬇ dftest-1.4-all.jar
New features
  • Source files dialog — the "Open file…" and "Add files…" toolbar buttons are replaced by a single 📂 Sources… button that opens a split-panel folder browser. The left panel navigates the filesystem; the right panel shows the currently loaded files. Supports adding individual files, all .java files in a folder, or recursively scanning a folder tree. Files can be removed individually or all at once.
  • Multiple test files — the test file dialog now manages a list of JUnit test files rather than a single file. All loaded test classes are instrumented and run together in one pass; traces from each test method are attributed correctly regardless of which class they belong to.
  • Test file folder scan — the test file dialog gains an "Add folder…" button that recursively scans a directory for .java files, mirroring the source file dialog.
Bug fixes
  • Allman-style brace instrumentation — tracer enter() and visit() calls are now correctly inserted inside the method body when opening braces appear on their own line, preventing misplaced calls that caused compilation failures on Allman-formatted source.
  • Local variable name collision — moved the internal tracer class to package cfgtracer. Previously, user source containing a local variable named cfg (e.g., a method parameter AnalysisConfig cfg) caused the instrumented call cfg.CFGTracer.visit(…) to be interpreted as member access, producing a compilation error.
  • Self-instrumentation guard — files loaded simultaneously as both source and test are no longer accidentally instrumented, preventing degenerate compilation loops.
  • Instrumentation errors to console — compilation errors from the instrumentation step are now printed to standard error with a full stack trace, rather than shown in a modal dialog that truncated long messages.
Version 1.3 Older
April 2026 ⬇ dftest-1.3-all.jar
New features
  • Coverage matrix — a new 📊 Matrix button opens a test × requirement table showing exactly which tests cover which DU-pairs and DU-paths. Columns can be marked as redundant or as part of a greedy minimal cover. A Combined column (shown when minimal cover is active) displays the union of coverage achieved by the cover subset.
  • Hierarchical satisfaction preference — within the coverage matrix, a Preference selector (Direct / Sidetrips / Detours) controls which tour quality counts as satisfying a requirement. The evaluation is per-requirement with fallback: a lower-quality tour is only accepted when no better tour exists for that specific requirement. Cells that were covered but fall below the effective threshold for their row are shown dimmed.
  • Method selector in matrix dialog — the coverage matrix can be switched to any other method in the source without closing the dialog.
  • Automatic coverage evaluation — coverage is now computed immediately whenever test paths arrive from any source (method switch, test execution, path click, or project load). The Check coverage button is only needed after manually editing the path field.
  • ✗ marks on empty test suite — DU-pairs and DU-paths panels now show ✗ for all requirements when the path field is empty and coverage has been checked, rather than leaving the panel blank.
Bug fixes
  • Spoon 11 / Java 21 upgrade — upgraded Spoon to 11.3.0 and set compliance level 21. Resolves PartialSourcePosition exceptions that occurred when analysing methods containing local record declarations or synthetic elements. Local records and comments are now silently skipped in the CFG builder.
Version 1.2 Older
April 2026 ⬇ dftest-1.2-all.jar
New features
  • Hierarchical satisfaction criteria — coverage evaluated at three levels: simple tour (✓), sidetrip (⍻), and detour ((⍻)). Status bar, panels, and HTML report show mode-aware marks and counts.
  • Multi-method test running — running a test file instruments all methods in a single pass; a progress indicator shows which method is being processed.
  • Graph overlay controls — zoom and node-size buttons shown directly on the graph panel.
  • Spoon compliance level — set to 17, fixing misparse of Java 16+ record declarations.
Version 1.1 Initial release
2025 ⬇ dftest.jar
Features
  • Interactive CFG construction from Java source via Spoon — handles loops, switches, try/catch/finally, break and continue
  • DU-pair and DU-path computation under all-uses and all-DU-paths criteria
  • Manual test path entry with check coverage and per-path graph highlighting
  • Live instrumentation — compiles and runs a JUnit test file, populates test paths from execution traces
  • Filter by variable, click-to-highlight source, click DU-path to import as test path
  • HTML coverage report across all methods
  • Project save/load (.cfgproject), configurable analysis and view settings
