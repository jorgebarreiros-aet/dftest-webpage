DFTest 1.2

Released April 2026  ·  dftest-1.2-all.jar  ·  requires JDK 21+

Adds hierarchical satisfaction criteria (simple / sidetrip / detour), multi-method test running, graph overlay zoom and node-size controls, and a Spoon compliance fix that resolves instrumentation errors with Java 16+ record declarations.

⬇  Download v1.2
Step 1 — Prerequisites

Make sure you have a JDK 21 (or later) installed. To check, run:

java -version

If not installed, download from Adoptium or any JDK 21 distribution.

Step 2 — Launch

Open a terminal in the folder where you saved the JAR and run:

java -jar dftest-1.2-all.jar

On Windows you can also double-click the JAR if your JDK is on the system PATH.

Step 3 — First use

Once the window opens:

  1. Paste or open a .java file
  2. Select a method from the dropdown
  3. Click ▶ Analyze
  4. Enter test paths and click Check coverage

To run test files automatically, use 📄 Set test file… to point DFTest at a JUnit 5 test class. The built-in preset button will locate JUnit jars in your local Maven repository automatically.

Release history
Version 1.2 Current
April 2026 ⬇ dftest-1.2-all.jar
New features
  • Hierarchical satisfaction criteria — coverage is now evaluated at three levels: simple tour (✓ ), tour with sidetrips (⍻ ), and tour with detours ((⍻)). The status bar, DU-pairs/paths panels, and HTML report all show mode-aware marks and counts. Sidetrip and detour sections are omitted when they add no coverage over the previous level.
  • Multi-method test running — running a test file now instruments and collects traces for all methods in the source in a single operation. A progress indicator in the status bar shows which method is currently being processed.
  • Graph overlay controls — zoom (+ − ⟳) and node size (◉− ◉+) buttons are now shown directly as an overlay on the graph panel. Node size is also still configurable via Settings.
Bug fixes
  • Spoon compliance level — the Spoon launcher was not configured with a Java source compliance level, causing record declarations (Java 16+) to be misparsed as method declarations. This produced spurious entries in the method selector and incorrect source-position reporting that caused instrumentation calls to land inside record parameter lists. Fixed by setting compliance level 17.
